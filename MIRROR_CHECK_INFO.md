# Зеркальный диапазон (Half-point mirror): Важная информация

## ⚠️ ВАЖНО: Это НЕ быстрая проверка!

### Реальность

**Размер диапазона 28-29% puzzle71:**
- ~1.18×10^19 ключей (1% от всего puzzle71)
- При CPU скорости ~1 Mkey/s: **тысячи лет**

**Это очень долгий процесс, не как проверка соседних ключей (±256)!**

---

## Стратегии

### Вариант 1: GPU на сервере (рекомендуется)

Зеркальный диапазон можно запустить на GPU параллельно с основным поиском:

```bash
# На сервере
cd ~/MyWalletSearch
./run_gpu_mirror.sh  # (нужно создать аналогично run_gpu_b9_wide.sh)
```

**Преимущества:**
- GPU в ~100-1000 раз быстрее CPU
- Работает параллельно с основным поиском
- Не блокирует локальные ресурсы

### Вариант 2: Узкий паттерн (если найден B9)

Если основной поиск найдёт адреса с `1PWo3JeB9*`, можно сузить паттерн до `1PWo3JeB9j` или `1PWo3JeB9jr`, что сильно уменьшит сложность.

### Вариант 3: Долгосрочная задача

Запустить как фоновую задачу на GPU и мониторить результаты периодически.

---

## Вывод

**Зеркальный диапазон (28-29%) — это долгосрочная задача, не быстрая проверка.**

**Рекомендация:** 
1. Сначала дождаться результатов основного поиска (CPU Step 1 + GPU широкий)
2. Если ничего не найдено — запустить зеркальный диапазон на GPU как дополнительную стратегию
