# GPU Step 1: Инструкции для сервера

## Что подготовлено

### 1. Широкий GPU поиск (рекомендуется запустить первым)

**Файлы:**
- `generate_segments_gpu_b9_wide.py` - генератор сегментов
- `seg_gpu_b9_wide.txt` - файл с 32 сегментами (уже создан)
- `run_gpu_b9_wide.sh` - скрипт запуска

**Параметры:**
- **Паттерн:** `1PWo3JeB` (8 символов) - GPU-фаза согласно анализу
- **Диапазон:** `2026000000000000000000` → `2038000000000000000000` (широкий, покрывает 71.5-72.5% + запас)
- **Сегментов:** 32
- **Grid:** `64,128` (RTX 4090)

### 2. Команды для запуска на сервере

```bash
cd ~/MyWalletSearch

# 1. Убедитесь что файл сегментов создан (или создайте на сервере)
python3 generate_segments_gpu_b9_wide.py > seg_gpu_b9_wide.txt

# 2. Запуск
./run_gpu_b9_wide.sh

# 3. Мониторинг
tail -f log_gpu_b9_wide.log
python3 show_segment_progress.py seg_gpu_b9_wide.txt progress_gpu_b9_wide.dat out_gpu_b9_wide.txt

# 4. Остановка
pkill -f 'VanitySearch.*-seg.*seg_gpu_b9_wide'
```

### 3. Преимущества параллельного запуска

**CPU (локально):**
- Паттерн: `1PWo3JeB9` (9 символов) - более узкий
- Узкие сегменты: ~6×10^15 каждый
- 2 экземпляра × 8 сегментов = 16 сегментов
- Фокус: центр ~2.0266×10^21

**GPU (сервер):**
- Паттерн: `1PWo3JeB` (8 символов) - более широкий
- Широкие сегменты: ~3.75×10^17 каждый
- 32 сегмента
- Фокус: весь диапазон 71.5-72.5% + запас

**Стратегия:**
- GPU быстро "прочешет" широкий диапазон с паттерном 8 символов
- CPU более точно ищет узкий паттерн 9 символов в фокусной зоне
- Оба работают параллельно, увеличивая шансы найти целевой адрес
