# Конфигурация сегментов для VanitySearch
# Формат: startPercent endPercent direction [name]
# 
# startPercent, endPercent: проценты от полного диапазона (0.0 - 100.0)
# direction: up (вверх) или down (вниз)
# name: имя сегмента (опционально)
#
# Пример для Puzzle 71 (рекомендуемый диапазон 48-55%):

# Сегмент 1: поиск вверх от 45% до 54%
45.0 54.0 up segment_up_45-54

# Сегмент 2: поиск вниз от 59% до 54%
59.0 54.0 down segment_down_59-54

# Дополнительные сегменты (раскомментируйте при необходимости):

# Сегмент 3: проверка области 48-50%
# 48.0 50.0 up hot_zone_1

# Сегмент 4: проверка области 52-55%
# 52.0 55.0 up hot_zone_2

# Сегмент 5: широкий поиск вниз
# 60.0 45.0 down wide_search

# ВАЖНО:
# - Каждый сегмент будет обрабатываться параллельно доступными потоками
# - Направление "up" означает поиск от меньших ключей к большим
# - Направление "down" означает поиск от больших ключей к меньшим
# - Сегменты могут пересекаться - это нормально для полного покрытия
