# Руководство по просмотру прогресса поиска

## Быстрый просмотр

```bash
./show_progress.sh
```

Показывает:
- Общий статус из лога
- Последнее сообщение о прогрессе по сегментам
- Количество проверенных ключей

## Детальный просмотр по сегментам

```bash
python3 show_segment_progress.py seg_gpu_range.txt progress_gpu_range.dat
```

Показывает таблицу с детальным прогрессом по каждому сегменту:

| Сегмент | Статус    | Прогресс % | Проверено  | Осталось    | Размер      | Ключей     |
|---------|-----------|------------|------------|-------------|-------------|------------|
| gpu1_1  | ▶ Активен | 15.23%     | 125.00T    | 697.50T     | 822.50T     | 1.2M       |
| gpu1_2  | ✓ Завершен| 100.00%    | 124.99T    | 0           | 124.99T     | 0.8M       |

### Параметры таблицы:
- **Сегмент** - имя сегмента из конфигурации
- **Статус** - Активен (▶) или Завершен (✓)
- **Прогресс %** - процент пройденного диапазона сегмента
- **Проверено** - сколько ключей уже проверено в этом сегменте (в абсолютных единицах)
- **Осталось** - сколько ключей осталось проверить до конца сегмента
- **Размер** - общий размер сегмента
- **Ключей** - количество проверенных ключей для этого сегмента (включая все варианты: point + endo + симметрии)

### Формат чисел:
- K = тысячи (×1000)
- M = миллионы (×1,000,000)
- G = миллиарды (×1,000,000,000)
- T = триллионы (×1,000,000,000,000)

## Синхронизация с параметрами запуска

Скрипт автоматически:
1. Читает **seg_gpu_range.txt** чтобы узнать границы сегментов (как они были заданы при запуске)
2. Читает **progress_gpu_range.dat** чтобы узнать текущие позиции (куда дошёл поиск)
3. Вычисляет прогресс относительно **исходных границ**, учитывая направление поиска (UP/DOWN)

Это означает что прогресс всегда считается относительно **тех параметров с которыми ты запускал поиск**.

## Примеры использования

### Посмотреть прогресс для текущего запуска
```bash
python3 show_segment_progress.py seg_gpu_range.txt progress_gpu_range.dat
```

### Посмотреть прогресс для другого запуска
```bash
python3 show_segment_progress.py seg_other.txt progress_other.dat
```

### В реальном времени (обновлять каждые 30 секунд)
```bash
watch -n 30 'python3 show_segment_progress.py seg_gpu_range.txt progress_gpu_range.dat'
```

## Требования

- Python 3.x
- Файл конфигурации сегментов (например, `seg_gpu_range.txt`)
- Файл прогресса (например, `progress_gpu_range.dat`) - создаётся автоматически при запуске с параметром `-progress`
